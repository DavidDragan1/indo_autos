System Requirements

Before proceeding with the installation, ensure that your environment meets the following prerequisites:

  • PHP: Version 8.0 or higher
  • Database: MySQL (ensure the required database system is installed and running).
  • Email Configuration: SMTP server setup is required for email notifications.
  • Web Server: Nginx or Apache must be installed and configured.

Backend Installation

  • Navigate to the Project Directory: Change to your web root directory where the backend will be installed.

            cd /var/www/

  • Create and Navigate to the Project Folder:
    • sudo mkdir your_project_folder 
    • cd your_project_folder
  • Upload Code: Upload your Laravel backend code to the your_project_folder directory using FTP, SCP, or any other file transfer method. Make sure all files are placed correctly in this folder.
  • Install Composer Dependencies: Once the backend code is uploaded, install the necessary dependencies using Composer.
    • composer install
  • Copy the Configuration File: Copy the applicaiton/config/config.example.php environment file to applicaiton/config/config.php.
    • cp applicaiton/config/config.example.php applicaiton/config/config.php
  • Copy the Database File: Copy the applicaiton/config/database.example.php environment file to applicaiton/config/database.php.
    • cp applicaiton/config/database.example.php applicaiton/config/database.php

 

  • Edit config.php File:

           Open the .env file and configure the following sections:

 

  • For Base url:
     $config['base_url'] = 'https://example.com/';

 

  • For Recaptcha:

           $config['site_key'] = '';

           $config['secret_key'] = '';

  • For SMTP MAIL 
    $config['email_host']               = '';
    $config['email_username']           = '';
    $config['email_password']           = '';
    $config['email_port']               = '';
    $config['from_name']                = '';
    $config['email_from']               = '';

 

  • Edit database.php File:            
   'hostname' => 'localhost',
   'username' => 'username',
   'password' => 'password',
   'database' => 'database',
  • Create Uploads Folder and Session Folder (If not exists):

            mkdir uploads
            mkdir application/sessions

  • Import Database:

           Import carquest.sql in your database

           using terminal: mysql -u user_name -p database_name < carquest.sql

Nginx Configuration:

If you are using Nginx as your web server, configure it to proxy requests to your backend:

  1. Create Nginx Configuration File:
    Open the Nginx configuration file for your backend: 
    sudo nano /etc/nginx/conf.d/backend.conf
  2. Paste the Following Configuration:
    Paste the following configuration in the file, replacing api.example.com with your domain or IP address: 
    server {
    server_name example.com;
    root /var/www/example.com/;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php8.1-fpm.sock; # Adjust PHP version if needed
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.ht {
        deny all;
    }

    error_page 404 /index.php;

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
  3. Test and Restart Nginx:
    After saving the configuration, test Nginx configuration and restart it: 
    sudo nginx -t
    sudo systemctl restart nginx

Additional Configuration:

  • Email Configuration: Ensure that your SMTP credentials are correctly set in the .env file for email functionality to work.
  • Firewall Configuration: Ensure that the necessary ports (usually 80, 443, 8000, 3000) are open in your firewall for both frontend and backend to function properly.
     

 

Obtain an SSL Certificate:

You can obtain an SSL certificate from:

  • Let's Encrypt (Free)
  • Commercial SSL Providers (Paid)

For Let's Encrypt, install Certbot and generate a free SSL certificate.

  1.  sudo snap install --classic certbot

  2.  sudo certbot --nginx
